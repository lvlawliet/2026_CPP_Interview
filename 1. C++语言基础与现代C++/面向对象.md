# 面向对象
## 虚函数表（vtable/vptr）  
### 1. 虚函数表是什么？  
虚函数表（vtable）是每个有虚函数的类在编译期生成的一张函数指针表，表中按声明顺序存放该类（及其继承链中）所有虚函数的地址。  
每个有虚函数的类（或其派生类）都会有一张独立的 vtable  
vtable 是静态的、全局的（通常放在只读数据段 .rodata）  
vtable 的内容在编译期确定，运行时不再改变  
### 2. 虚指针（vptr）是什么？  
vptr 是对象实例中隐藏的一个指针（通常是第一个成员），它指向该对象真实类型所对应的 vtable。  
只有含有虚函数的类（或继承自含有虚函数的类）的对象才会有 vptr  
vptr 的大小通常是 *sizeof(void)**（64位系统一般 8 字节）  
vptr 在构造函数中被初始化（最先被设置）  
### 3. 虚函数调用流程
``` text
普通成员函数调用：    obj.func()  →  直接跳到函数地址（编译期确定）

虚函数调用：          obj.func()  →  
                      1. 通过 obj 中的 vptr 找到 vtable
                      2. 根据虚函数在类中声明的顺序（索引）找到对应槽位
                      3. 取出该槽位的函数指针
                      4. 调用该函数指针（this 指针作为隐式参数传递）
```  
### 4. 什么时候产生 vtable？  
| 情况                               | 是否产生 vtable | 是否产生 vptr | 说明                                          |
|------------------------------------|------------------|----------------|-----------------------------------------------|
| 类中完全没有虚函数                 | 否               | 否             |                                               |
| 类中有虚函数（包括纯虚函数 =0）    | 是               | 是             | 即使是抽象类也有 vtable                       |
| 派生类重写了基类的虚函数           | 是（新表）       | 是             | 派生类有自己的 vtable                         |
| 派生类新增了虚函数                 | 是（新表）       | 是             |                                               |
| 类有虚函数，但对象是栈上/全局变量  | 是               | 是             | vptr 依然存在                                 |
| 使用 final / override 关键字       | 仍然产生         | 仍然产生       | final 只限制进一步重写，不影响 vtable 生成   |  
### 5. 典型内存布局示例（64 位系统）  
``` cpp
class Base {
public:
    virtual void f1() {}
    virtual void f2() {}
    virtual void f3() {}
    int x = 10;
};

class Derived : public Base {
public:
    void f2() override {}           // 重写
    virtual void f4() {}            // 新增
    int y = 20;
};
```
Base 对象内存布局（忽略对齐影响）：  
``` text
偏移 0x00: vptr    → 指向 Base::vtable
偏移 0x08: x       (4字节 + 4字节 padding)
```
Derived 对象内存布局：  
``` text
偏移 0x00: vptr    → 指向 Derived::vtable
偏移 0x08: x
偏移 0x0c: padding
偏移 0x10: y
```  
vtable 内容示意（伪代码）：  
``` text
Base::vtable:
  [0] &Base::f1
  [1] &Base::f2
  [2] &Base::f3

Derived::vtable:
  [0] &Base::f1      （未重写，继承）
  [1] &Derived::f2   （重写）
  [2] &Base::f3      （未重写）
  [3] &Derived::f4   （新增）
```
### 6. 高频问题   
| 问题                                       | 标准回答要点                                                                 |
|--------------------------------------------|-------------------------------------------------------------------------------|
| 虚函数表是每个对象一张吗？                 | 不是。**每个类**一张 vtable，**每个对象**一个 vptr 指向对应类的 vtable。      |
| vtable 存放在内存的哪个区域？              | 通常在**只读数据段**（.rodata），是静态的、全局的。                           |
| 构造函数中可以调用虚函数吗？               | 可以，但**只会调用当前类的版本**（因为 vptr 还未指向最终类型）。              |
| 析构函数中可以调用虚函数吗？               | 可以，但**从当前类向上调用**，调用的是当前析构阶段的版本（非常危险，极少使用）。 |
| 虚函数表一共有几张？                       | 继承链上每个**引入或修改过虚函数**的类都会有一张独立的 vtable。               |
| 多重继承下 vtable 如何组织？               | 每个基类可能有独立的 vtable，对象中会有多个 vptr（实现较复杂，面试较少深挖）。 |
| final / override 对 vtable 有影响吗？      | 没有影响，只是编译期限制，不改变 vtable 的生成和使用。                        |
| 虚函数调用一定通过 vtable 吗？             | 大多数情况下是，但编译器可能优化（devirtualization）：当类型明确时可直接调用。 |  
### 7. 经典代码实验题  
``` cpp
struct Base {
    virtual void who() { std::cout << "Base\n"; }
};

struct A : Base {
    void who() override { std::cout << "A\n"; }
};

struct B : Base {
    void who() override { std::cout << "B\n"; }
};

int main() {
    Base* p1 = new A;
    Base* p2 = new B;
    Base  b;

    p1->who();
    p2->who();
    b.who();

    delete p1;
    delete p2;
}
```
输出：  
``` text
A
B
Base
```  
### 多继承的vptr布局  
链式单继承  
``` cpp
struct C {                          // 最终基类
    virtual void fC() { std::cout << "C::fC\n"; }
    int c = 1;
};

struct B : public C {               // 中间层
    virtual void fB() { std::cout << "B::fB\n"; }
    int b = 2;
};

struct A : public B {               // 最派生类
    virtual void fA() { std::cout << "A::fA\n"; }
    int a = 3;
};
```
内存布局图  
``` text
偏移量     内容                          大小     说明
0x00      vptr (A 的完整虚表)           8 字节   指向最终虚表（fC/fB/fA 全在这里）
0x08      C::c (int)                    4 字节   基类成员最先
0x0C      B::b (int)                    4 字节   中间层成员
0x10      A::a (int)                    4 字节   派生类成员
0x14      padding（填充）               4 字节   让整个对象按 8 字节对齐
────────────────────────────────────
总大小：sizeof(A) = 24 字节
```  
A的虚表内容  
``` text
[0]  &A::fC     （或 override 后的版本）
[1]  &A::fB     （或 override 后的版本）
[2]  &A::fA
```  
多层嵌套多继承  
``` cpp
#include <iostream>
#include <cstddef>

struct D { virtual void fd() {} int d = 1; };
struct E { virtual void fe() {} int e = 2; };
struct B : public D, public E { virtual void fb() {} int b = 3; };

struct F { virtual void ff() {} int f = 4; };
struct G { virtual void fg() {} int g = 5; };
struct C : public F, public G { virtual void fc() {} int c = 6; };

struct A : public B, public C { virtual void fa() {} int a = 7; };

int main() {
    A obj;
    std::cout << "sizeof(A) = " << sizeof(A) << " 字节\n";
    std::cout << "D* 偏移 = " << ((char*)(D*)&obj - (char*)&obj) << "\n";  // 0
    std::cout << "E* 偏移 = " << ((char*)(E*)&obj - (char*)&obj) << "\n";  // 16
    std::cout << "F* 偏移 = " << ((char*)(F*)&obj - (char*)&obj) << "\n";  // 40
    std::cout << "G* 偏移 = " << ((char*)(G*)&obj - (char*)&obj) << "\n";  // 56
    std::cout << "C* 偏移 = " << ((char*)(C*)&obj - (char*)&obj) << "\n";  // 40
}
```  
内存布局图  
``` text
偏移量   内容                    大小     说明
0x00     vptr (D Primary)        8 字节   主虚表（A/B/D 全路径虚函数都在这里）
0x08     D::d (int)              4 字节
0x0C     padding                 4 字节   对齐下一个 vptr
0x10     vptr (E)                8 字节   B 内部的 Secondary vptr
0x18     E::e (int)              4 字节
0x1C     padding                 4 字节
0x20     B::b (int)              4 字节
0x24     padding                 4 字节   ← B 子对象结束（共 40 字节）
────────────────────────────────────
0x28     vptr (F)                8 字节   C 子对象开始（A 的 Secondary vptr）
0x30     F::f (int)              4 字节
0x34     padding                 4 字节
0x38     vptr (G)                8 字节   C 内部的 Secondary vptr
0x40     G::g (int)              4 字节
0x44     padding                 4 字节
0x48     C::c (int)              4 字节
0x4C     padding                 4 字节   ← C 子对象结束（共 40 字节）
────────────────────────────────────
0x50     A::a (int)              4 字节
0x54     padding                 4 字节   对齐整个对象
────────────────────────────────────
总大小：88 字节
```
## 多态 / 继承 / 虚继承 / 菱形继承  
### 1. 多态（Polymorphism）详解  
多态分为两大类：  
| 类型         | 名称           | 实现方式               | 绑定时机     | 典型例子                  | 性能       |
|--------------|----------------|------------------------|--------------|---------------------------|------------|
| **静态多态** | 编译期多态     | 函数重载、模板、CRTP   | 编译期       | `std::max`、模板函数      | 极快       |
| **动态多态** | 运行时多态     | 虚函数 + 继承          | 运行期       | 基类指针调用派生类方法    | 稍慢（虚表查找） |  
动态多态核心机制（虚函数表 vtable + vptr）：  
基类指针/引用指向派生类对象时，调用虚函数会通过vptr → vtable 找到实际函数地址  
这就是“父类指针调用子类方法”的本质  
### 2. 继承类型对比  
| 继承方式     | 语法                              | 子类是否包含基类成员       | 典型用途                     | 主要问题 |
|--------------|-----------------------------------|----------------------------|------------------------------|----------|
| 单继承       | `class D : public B`              | 是                         | 普通继承                     | 无       |
| 多继承       | `class D : public B1, public B2`  | 是（各自一份）             | 接口继承、混合继承           | 菱形继承问题 |
| 虚继承       | `class D : virtual public B`      | 是（仅共享一份）           | 解决菱形继承                 | 构造顺序复杂、访问稍慢 |  
3. 菱形继承（Diamond Problem）  
场景：  
``` text
      A
     / \
    B   C
     \ /
      D
```
``` cpp
struct A { int x = 10; };
struct B : A {};
struct C : A {};
struct D : B, C {};   // 菱形继承
```
会出现两个严重问题：  
二义性（Ambiguity）  
``` cpp
D d;
d.x = 100;        // 错误！A::x 有两份，编译器不知道用哪一个
```
重复子对象（重复内存）  
D 对象里会有两份 A 子对象（B 里一份 + C 里一份）  
sizeof(D) = sizeof(A) * 2 + sizeof(B/C 其他成员)  
内存布局（普通多继承）：  
``` text
D 对象：
├── B 子对象
│   └── A 子对象 (x=10)
├── C 子对象
│   └── A 子对象 (x=10)   ← 重复！
```
### 4. 虚继承（Virtual Inheritance）彻底解决菱形问题
``` cpp
struct A { int x = 10; };
struct B : virtual public A {};   // 关键：virtual
struct C : virtual public A {};
struct D : B, C {};               // 现在只有一份 A
```  
解决效果：  
A 只存在一份（共享基类）  
d.x 没有二义性  
sizeof(D) 大幅减小  
  
虚继承的底层实现：  
编译器为每个虚基类生成一个虚基指针（vbptr）  
对象中多出一个或多个 vbptr，指向虚基类偏移表（vbase table）  
通过 vbptr + 偏移表 找到唯一的虚基类子对象  
  
内存布局对比  
``` text
D 对象：
├── B 子对象 (含 vptr)
│   └── A 子对象 (x)
├── C 子对象 (含 vptr)
│   └── A 子对象 (x)   ← 重复！
sizeof(D) ≈ 2×sizeof(A) + 开销

**虚继承（无菱形）**：
D 对象：
├── B 子对象 (含 vbptr)
├── C 子对象 (含 vbptr)
└── 共享的 A 子对象 (x)   ← 只有一份！
sizeof(D) ≈ sizeof(A) + 2×vbptr 大小
```
### 5. 虚继承下的构造函数调用顺序  
规则：  
先调用所有虚基类的构造函数（按继承声明顺序，从最左到最右）  
再按普通继承顺序调用非虚基类的构造函数  
最后调用派生类自身的构造函数  
示例：  
``` cpp
struct D : B, C { D() { std::cout << "D\n"; } };
```
调用顺序永远是：  
A() → B() → C() → D()   
### 6. 虚继承的优缺点
优点：  
彻底解决菱形继承的二义性和重复子对象问题  
节省内存（只有一份基类子对象）  

缺点：  
访问虚基类成员需要两次间接寻址（vbptr → 偏移表 → 基类），性能略有损失  
构造函数调用顺序复杂，容易出错  
对象大小增加（多了 vbptr）  
  
现代 C++ 建议：  
能用接口（纯虚类）就别用多继承  
必须多继承时，优先用虚继承解决菱形问题  
性能敏感场景考虑组合优于继承或 CRTP 静态多态  

### 7. 面试最常被问的 8 个问题  
菱形继承为什么会出现二义性？  
→ 因为同一基类被继承两次，产生了两份子对象。  
虚继承如何解决？  
→ 通过虚基指针（vbptr）让所有派生路径共享同一份虚基类子对象。  
虚继承会增加对象大小吗？  
→ 会（多 vbptr），但会减少重复基类占用的空间，总大小通常更小。  
虚继承下构造函数调用几次？  
→ 虚基类只构造一次，由最终派生类负责调用。  
没有虚继承时，基类指针能否指向派生类？  
→ 可以，但多继承时偏移计算复杂（需要调整 this 指针）。  
虚函数 + 虚继承会怎么样？  
→ 更复杂，会出现多个 vptr + vbptr，面试很少深挖。  
C++ 标准库里哪里用了虚继承？  
→ 极少，std::exception 体系曾用过，但现代标准库基本避免多继承。  
多继承时如何避免菱形问题？  
→ 用虚继承；或者改用接口 + 组合模式。
