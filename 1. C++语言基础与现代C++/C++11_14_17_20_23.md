##  C++11  
### 1. auto 类型推导 + decltype  
背景：减少重复类型声明（如复杂迭代器、lambda返回类型）。  
``` cpp
auto x = 42;                    // int
auto& ref = x;                  // int&
const auto* p = &x;             // const int*
decltype(x) y = x;              // int（保留cv和引用）
decltype((x)) z = x;            // int&（加括号视为左值表达式）
```
优势：简化代码、可读性、与模板完美配合。  
注意/陷阱：必须初始化；不能用于函数形参（C++14后lambda可）；auto推导会decay（丢引用/顶层const）；decltype((var))保留左值性。  
边缘：auto&& 为universal reference（完美转发关键）。  
### 右值引用、移动语义、完美转发  
``` cpp
class Widget {
    std::string data;
public:
    Widget(Widget&& other) noexcept : data(std::move(other.data)) {}  // 移动构造
    Widget& operator=(Widget&&) noexcept { /* 同上 */ }
};

// 完美转发
template<typename T>
void wrapper(T&& arg) {  // T&& 是universal reference
    foo(std::forward<T>(arg));  // 保留左/右值性
}
```
std::move：仅static_cast<T&&>，告诉编译器“可偷资源”。  
优势：性能飞跃（零拷贝转移）；配合Rule of Five（构造/析构/拷贝/移动/赋值）。  
注意/陷阱：移动后原对象必须合法（置空/有效状态）；不要对const对象move；small string optimization下move可能不优于拷贝；vector扩容时noexcept move ctor才能触发优化。  
边缘：引用折叠规则（T& & → T&，T&& && → T&&）；std::move_if_noexcept。
