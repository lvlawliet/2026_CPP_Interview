# C++11
## 一、新增关键词列表
alignas、alignof  
char16_t、char32_t  
constexpr  
decltype  
noexcept  
nullptr  
static_assert  
thread_local  
  
上下文关键词（特定场景下特殊含义）：  
final、override  
  
重获新义：auto（原存储类说明符 → 类型自动推导）。  
此外，default 和 delete 在特殊成员函数中新增用法。下面逐个讲解。  
### 1. auto —— 类型自动推导
解释：编译器根据初始化表达式自动推断变量类型，简化模板和迭代器代码。不能单独使用（如 auto x; 错误），也不能用于函数参数（C++14 后放开）。  
``` cpp
#include <iostream>
#include <vector>

int main() {
    auto i = 42;                    // int
    auto d = 3.14;                  // double
    auto s = "hello";               // const char*
    
    std::vector<int> v{1, 2, 3};
    for (auto& elem : v) {          // 常与范围 for 配合
        std::cout << elem << " ";
    }
    std::cout << std::endl;
    return 0;
}
```
### 2. decltype —— 表达式类型推导
解释：根据表达式（不求值）推导类型，常用于模板返回值或复杂类型。比 auto 更精确，可处理引用、const 等修饰。  
``` cpp
#include <iostream>

int main() {
    int x = 5;
    decltype(x) y = x;              // y 的类型为 int
    
    const int& ref = x;
    decltype(ref) z = x;            // z 的类型为 const int&
    
    // 模板中常用（C++11 尾随返回类型）
    auto add = [](auto a, auto b) -> decltype(a + b) {
        return a + b;
    };
    std::cout << add(1, 2.5) << std::endl;  // 3.5
    return 0;
}
```
### 3. nullptr —— 空指针字面量
解释：专属空指针类型（std::nullptr_t），解决 NULL（实际是 0）导致的重载歧义和类型安全问题。
```cpp
#include <iostream>

void foo(int* p) { std::cout << "指针版本\n"; }
void foo(int i)  { std::cout << "int 版本\n"; }

int main() {
    foo(nullptr);   // 明确调用指针版本
    // foo(NULL);   // 编译错误或歧义（取决于实现）
    return 0;
}
```
### 4. constexpr —— 常量表达式
解释：要求在编译期求值，可用于变量、函数、构造函数。提升性能，支持 static_assert 等。
``` cpp
#include <iostream>

constexpr int factorial(int n) {
    return n <= 1 ? 1 : n * factorial(n - 1);
}

int main() {
    constexpr int f = factorial(5);     // 编译期算出 120
    std::cout << "5! = " << f << std::endl;
    
    static_assert(factorial(5) == 120, "计算错误");
    return 0;
}
```
### 5. static_assert —— 编译期断言
解释：编译时检查条件，失败则报错（带自定义消息）。常用于模板约束。
``` cpp
#include <type_traits>

static_assert(sizeof(int) >= 4, "int 必须至少 4 字节");

template<typename T>
void check_type() {
    static_assert(std::is_integral<T>::value, "T 必须是整数类型");
}

int main() {
    check_type<int>();     // 成功
    // check_type<double>(); // 编译失败
    return 0;
}
```
### 6. noexcept —— 不抛异常说明符
解释：声明函数保证不抛异常，编译器可优化（栈展开、move 语义判断）。noexcept(true/false) 可动态指定。
``` cpp
#include <iostream>

void safe_func() noexcept {
    // 保证不抛异常
}

void may_throw() noexcept(false) {}

int main() {
    std::cout << std::boolalpha
              << "safe_func noexcept? " << noexcept(safe_func()) << "\n"  // true
              << "may_throw noexcept? " << noexcept(may_throw()) << std::endl; // false
    return 0;
}
```
### 7. thread_local —— 线程本地存储
解释：每个线程拥有独立副本，类似 static 但线程隔离。常用于多线程计数器。
``` cpp
#include <iostream>
#include <thread>

thread_local int tls_counter = 0;

void task(int id) {
    tls_counter += id;
    std::cout << "线程 " << id << " 的 tls_counter = " << tls_counter << std::endl;
}

int main() {
    std::thread t1(task, 1);
    std::thread t2(task, 2);
    t1.join();
    t2.join();
    return 0;
}
```
### 8. alignas / alignof —— 内存对齐控制
解释：alignas 指定对齐字节，alignof 查询对齐要求。用于 SIMD、缓存优化。
``` cpp
#include <iostream>

struct alignas(16) Aligned {
    int data[4];
};

int main() {
    std::cout << "Aligned 对齐要求: " << alignof(Aligned) << " 字节" << std::endl;  // 16
    return 0;
}
```
### 9. char16_t / char32_t —— Unicode 字符类型
解释：原生支持 UTF-16/UTF-32（配合 u""、U"" 字面量）
``` cpp
char16_t c16 = u'中';
char32_t c32 = U'界';
```
### 10. final / override —— 虚函数精确控制
解释：override 显式覆盖基类虚函数（编译器检查）；final 禁止派生类覆盖或继承。
``` cpp
class Base {
public:
    virtual void foo() { std::cout << "Base\n"; }
    virtual void bar() {}
};

class Derived final : public Base {  // final 类不可被继承
public:
    void foo() override { std::cout << "Derived\n"; }  // 必须写 override
    // void bar() final;  // 可在成员上加 final
};

int main() {
    Derived d;
    d.foo();
    return 0;
}
```
### 11. =default / =delete —— 特殊成员函数控制
解释：显式要求编译器生成默认实现，或禁止拷贝/赋值。
``` cpp
class NoCopy {
public:
    NoCopy() = default;
    NoCopy(const NoCopy&) = delete;
    NoCopy& operator=(const NoCopy&) = delete;
};
```
## 二、其他重要新特性
### 1.Lambda 表达式（匿名函数，支持捕获）
| 写法          | 含义                                           | 生命周期注意事项               |
|:-------------:|:-----------------------------------------------|:-------------------------------|
| `[]`          | 不捕获任何外部变量                             | -                              |
| `[=]`         | **值捕获** 所有用到的外部变量                  | 复制一份，安全                 |
| `[&]`         | **引用捕获** 所有用到的外部变量                | 易悬挂引用！危险               |
| `[x, y]`      | 值捕获指定的变量 x 和 y                        | -                              |
| `[&x, &y]`    | 引用捕获指定的变量 x 和 y                      | -                              |
| `[=, &x]`     | 所有值捕获，但 x 使用引用捕获                  | 常用混合写法                   |
| `[&, x]`      | 所有引用捕获，但 x 使用值捕获                  | 常用混合写法                   |
| `[this]`      | 捕获当前对象的 `this` 指针                     | 成员函数内常用                 |
| `[=,*this]`   | C++17：值捕获整个对象（C++11 不支持）          | -                              |
| `[&,*this]`   | C++17：引用捕获整个对象（C++11 不支持）        | -                              |
``` cpp
int main() {
    int a = 10, b = 20;

    // 1. 值捕获（推荐，安全）
    auto lambda1 = [=]() { return a + b; };          // a、b 被复制

    // 2. 引用捕获（小心生命周期！）
    auto lambda2 = [&](int x) { a += x; return a; };

    // 3. 混合捕获（最常用）
    auto lambda3 = [=, &b](int x) { return a * x + b; };

    // 4. 捕获 this（类成员函数内）
    class Test {
    public:
        int val = 42;
        auto getLambda() {
            return [this]() { return val; };     // 捕获 this 指针
            // return [=]() { return val; };     // C++11 推荐写法（值捕获 this）
        }
    };
}
```
mutable 关键字  
默认情况：Lambda 是 const 的，值捕获的变量不能修改。  
``` cpp
int counter = 0;
auto bad = [=]() {
    // counter++;     // 编译错误！Lambda 是 const
};

auto good = [=]() mutable {   // 加 mutable 后可修改副本
    counter++;
    return counter;
};
```

# C++14
## 一、语言特性
### 1. 函数返回值类型推导
解释：C++11 只允许 lambda 和模板函数的 auto 返回类型推导；C++14 扩展到普通函数。编译器根据 return 语句自动推导返回类型。支持多 return（类型必须一致），但递归函数需至少一个先导 return。  
新增语法：auto + decltype(auto)（C++14 新增，支持完美保留引用/值类别）。  
``` cpp
#include <iostream>

// 普通函数
auto add(int a, int b) {  // 无需写 -> int
    return a + b;
}

// decltype(auto) 版本（保留引用）
template<typename T>
decltype(auto) forward_like(T&& x) {
    return std::forward<T>(x);  // 完美转发
}

int main() {
    std::cout << add(10, 20) << std::endl;  // 30
    int n = 42;
    auto& ref = forward_like(n);  // ref 是 int&
    std::cout << ref << std::endl;
}
```
### 2. 泛型 Lambda 表达式
解释：lambda 参数可使用 auto，闭包的 operator() 自动变成函数模板，实现泛型（类似模板函数）。极大简化泛型回调代码。  
``` cpp
#include <iostream>
#include <vector>

int main() {
    auto add = [](auto a, auto b) { return a + b; };  // 泛型！

    std::cout << add(1, 2) << std::endl;        // 3 (int)
    std::cout << add(1.5, 2.5) << std::endl;    // 4 (double)
    std::cout << add(std::string("Hello"), " World") << std::endl;

    // 带参数包的泛型 lambda（C++14 支持）
    auto print = [](auto printer, auto&&... args) {
        (printer(args), ...);
    };
    print([](auto x){ std::cout << x << " "; }, 1, 2.5, "hi");
}
```
这里(printer(args), ...);的...是17的折叠写法。
### 3. Lambda 初始化捕获
解释：捕获列表中可直接初始化新变量，支持移动（move-only 类型）和表达式计算，解决 C++11 中捕获临时对象/移动的痛点。  
``` cpp
#include <memory>
#include <iostream>

int main() {
    auto ptr = std::make_unique<int>(42);

    // 初始化捕获 + 移动
    auto lambda = [p = std::move(ptr)]() {
        std::cout << *p << std::endl;  // 42
    };
    lambda();  // ptr 已移走

    // 引用初始化捕获示例
    int x = 4;
    auto y = [&r = x, x = x + 1]() -> int {
        r += 2;        // 修改原 x
        return x * x;  // 使用捕获的新 x（5）
    }();
    std::cout << y << std::endl;  // 25
}
```
### 4. 变量模板
解释：允许模板化变量（尤其是 constexpr 常量），取代 C++11 中只能用模板结构体存常量的笨拙方式。  
``` cpp
#include <iostream>

// 变量模板
template<class T>
constexpr T pi = T(3.1415926535897932385L);

template<class T>
T circular_area(T r) {
    return pi<T> * r * r;
}

int main() {
    std::cout << pi<int> << std::endl;           // 3
    std::cout << pi<double> << std::endl;        // 3.14159...
    std::cout << circular_area(5.0) << std::endl; // 78.5398
}
```
### 5. constexpr 函数限制大幅放宽
解释：C++11 constexpr 函数只能有一条 return 且无局部变量；C++14 允许局部变量、if、for、while、多条 return（仍不能动态分配、goto、try-catch 等）。
``` cpp
constexpr int factorial(int n) {
    int res = 1;
    for (int i = 2; i <= n; ++i) {
        res *= i;
    }
    return res;
}

int main() {
    static_assert(factorial(5) == 120, "");  // 编译期计算
    constexpr int f = factorial(10);
    std::cout << f << std::endl;
}
```
### 6. 二进制字面量 + 数字分隔符
解释：  
二进制：0b / 0B 前缀。  
分隔符：'（单引号）提高大数字可读性（不影响值）。  
``` cpp
int main() {
    int bin = 0b1010'1010;     // 170（二进制）
    long long big = 1'000'000'000'000LL;
    double d = 3.141'592'6535;

    std::cout << bin << " " << big << std::endl;
}
```
### 7. [[deprecated]] 属性
解释：标记函数、类、变量为“已废弃”，编译时发出警告（可带消息）。不是关键字，是标准属性。
``` cpp
[[deprecated("Use new_func() instead")]]
void old_func() {}

struct [[deprecated]] OldStruct {};

int main() {
    old_func();        // 警告
    OldStruct s;       // 警告
}
```
## 二、标准库特性
### 1. std::make_unique
解释：C++11 有 make_shared，C++14 补齐 make_unique，避免手动 new 的异常安全问题。
```cpp
#include <memory>
auto p1 = std::make_unique<int>(42);
auto p2 = std::make_unique<int[]>(10);  // 数组版本
```
### 2. std::exchange
解释：原子式交换值并返回旧值，常用于状态机。  
``` cpp
#include <utility>
int old = std::exchange(value, 100);  // value 变为100，返回原 value
```
### 3. std::quoted
``` cpp
#include <iomanip>
std::cout << std::quoted("hello world") << std::endl;  // 输出 "hello world"
```
### 4. std::integer_sequence
示例：用于解包参数包、索引序列等（C++14 引入）。
### 5. std::shared_timed_mutex + std::shared_lock（读写锁）
解释：支持多读单写 + 超时。
``` cpp
#include <shared_mutex>
std::shared_timed_mutex mtx;
int value;

void read() {
    std::shared_lock lock(mtx);  // 多线程可同时读
    // ...
}

void write(int v) {
    std::lock_guard lock(mtx);   // 独占写
    value = v;
}
```
